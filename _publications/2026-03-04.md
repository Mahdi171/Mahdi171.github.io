date: 2026-03-04

---

<ul>
<li>Quantum computers threaten blockchains that rely on <strong>ECDSA</strong>, since exposed public keys may allow adversaries to recover private keys using quantum algorithms.</li>

<li>Existing post-quantum migration proposals often require <strong>address changes</strong>, hybrid signature schemes, or mechanisms that fail to protect <strong>inactive accounts</strong>.</li>

<li>We observe that blockchains using <strong>EdDSA</strong> signatures (e.g., Sui, Solana, Near) have a structural advantage: their keys are deterministically derived from a <strong>seed</strong> according to <strong>RFC 8032</strong>.</li>

<li>This property allows users to prove ownership of an account by proving knowledge of the <strong>EdDSA seed</strong> in <strong>zero knowledge</strong>, without revealing elliptic-curve secrets.</li>

<li>We introduce a <strong>post-quantum migration protocol</strong> where the EdDSA seed acts as the witness in a zero-knowledge proof that authorizes new <strong>quantum-safe signatures</strong>.</li>

<li>The protocol enables <strong>seamless migration</strong> to post-quantum security while preserving existing addresses and maintaining backward compatibility.</li>

<li>Our approach works even when <strong>public keys have already been exposed</strong>, which is common for active blockchain accounts.</li>

<li>We formalize a <strong>security model for post-quantum ready signatures</strong>, capturing both dual-mode security (classical and post-quantum) and compatibility with existing blockchain systems.</li>

<li>We prove the security of the construction using a <strong>game-based argument</strong> showing that knowledge of the EdDSA seed is necessary to authorize the migration.</li>

<li>We implement a proof-of-concept using the <strong>Ligetron zkVM</strong>, demonstrating practical performance with <strong>6.2s proving time, 2.3s verification time, and 5.4MB proof size</strong>.</li>
</ul>

Tiny side note from the cryptography nerd corner: the trick of using the EdDSA seed instead of the scalar is what makes the whole thing elegant. RFC 8032 quietly gave EdDSA chains a time capsule for post-quantum migration years before anyone was worrying about it seriously. Sometimes standards hide little gifts for the future.
